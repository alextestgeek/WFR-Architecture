# Experiment 07 — WFRA-2.0 / RFP v1 **прекурсор** (композитный лосс + warm-up)

## Зачем этот эксперимент (не «как у обычной LLM»)

Здесь модель — **резонансное поле + readout**, а не стандартный Transformer. Смысловые метрики — **RC, фазы, спайковая активность**, а не только perplexity на огромном корпусе. Поэтому:

- **Протокол 06-rfp-protocol-tests** остаётся инженерным: свежие train-батчи, tier A/B/C, «ядро не ломается».
- **Эксперимент 07** добавляет слой **соответствия теории**: явные \(\mathcal{L}_{\text{res}}, \mathcal{L}_{\text{energy}}, \mathcal{L}_{\text{phase}}\) в общем лоссе и **warm-up только readout + контентные фазы** — как в твоём плане этапа 1.

Полный **RFP v1 без глобального backward** и **32 уровня с learnable coupling** в этом каталоге **не реализованы** — это следующий шаг после того, как композитный лосс покажет стабильное поведение в autograd.

## Правки к твоему последнему плану (сжато)

| Было в плане | Уточнение |
|--------------|-----------|
| «32 уровня» | В текущем `WFRNetwork` — **4** резонансных слоя; `num_fractal_levels=6` — не то же самое, что «уровни RFP». Числа в доке нужно привязать к тензорам. |
| \(\partial\mathcal{L}_{\text{task}}/\partial\text{logits}\) как `error` на уровне \(l\) | Это сигнал **выхода**; локальная ошибка на слое — **отдельное определение** (проекция / аппроксимация). В прекурсоре RFP по слоям не трогаем. |
| \(\mathcal{L}_{\text{res}}\) по слоям \(l\) | Сейчас **одна** сетка фаз `phases` [B,T,P] на все слои → формула сводится к **одному** порядковому параметру по группам фаз (см. `composite_loss.py`). |
| «Без глобального backward» на этапе 2 | Пока в лосс входят члены от `phases` с autograd, **backward есть**. Отключение — только с `detach` + отдельные локальные правила (будущий RFP v1). |
| \(C_l\) coupling | В ядре **нет** отдельного learnable coupling между слоями — это **новая архитектура**, не только тренер. |

## Что сделано в коде

1. **`wfr_lm.py`**: в `WFRState` проброшены **`phases`** из выхода ядра (раньше было `None`). Ядро не менялось.
2. **`composite_loss.py`**:  
   \(\mathcal{L}_{\text{total}} = \mathcal{L}_{\text{task}} + \alpha\mathcal{L}_{\text{res}} + \beta\mathcal{L}_{\text{energy}} + \gamma\mathcal{L}_{\text{phase}}\)  
   с реализациями, согласованными с текущими тензорами.
3. **`train_wfra2_precursor.py`**: warm-up (заморозка **core**, обучение **head + token_phase_offset**), затем полный AdamW; **два прогона подряд**: `ce_only` vs `wfra2_composite`; **fresh train / fixed val** как в протоколе 06.

## Что ожидаем увидеть

- **Не** обязательно «лучше CE, чем у GPT» — микроскопический vocab и toy.
- **Да**: стабильное обучение (градиенты finite), сравнимые кривые val CE; иногда композитный лосс даст **другой баланс** поля (RC/spike), что видно по косвенным метрикам в логах частей лосса.
- Если композитный лосс **ухудшает** CE — это тоже **полезный результат**: значит веса \(\alpha,\beta,\gamma\) или определения членов нужно калибровать под масштабы — это нормальный этап для не-LLM.

## Запуск

Из корня репозитория:

```powershell
python experiments/07-wfra2-precursor/train_wfra2_precursor.py --epochs 24 --warmup-epochs 8
python -m pytest experiments/07-wfra2-precursor/test_composite_loss.py -v
```

Удалённый GPU: как в `RULES.md`, подставить путь `experiments/07-wfra2-precursor/`, `PYTHONPATH=.` от корня `WFR-Memory-Test`, плюс обновлённые `wfr_lm.py` в корне копии.

## Связь с 06-rfp-protocol-tests

Протокол **06** не заменён: он проверяет **устойчивость и tier-ы**. **07** проверяет **гипотезу о композитном лоссе** в той же духе «свой путь от теории», без требования «как у обычной LLM».
