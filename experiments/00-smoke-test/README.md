**Версия:** 2.1 (Theoretical Exploration)

**Статус:** Завершён (рефакторинг + анализ)  
**Ветка:** `experiment/01-long-context`

## Гипотеза

Проверить, насколько базовая архитектура WFR может быть реализована **точно по теории** (Wave Phase Encoder + PhaseInterference + TheoreticalResonanceLayer).

## Что было сделано

- Реализован `PhaseInterference` согласно формулам `R = cos(Δφ)` и `Re(∑ w_j · e^{i(φ_j - φ_target)})`
- Создан `TheoreticalResonanceLayer`
- Протестировано 3 стратегии выбора `φ_target`: `mean`, `frequency`, `base`
- Добавлено логирование параметров слоёв

## Результаты (лучший вариант — frequency)

**Context 512:**
- RC: 0.9281
- Spike rates: 60.4%, 0.0%, 0.0%, 25.8%

**Context 2048:**
- RC: 0.8229
- Spike rates: 26.8%, 0.0%, 0.0%, 31.4%

**Context 8192:**
- RC: 0.7010
- Spike rates: 16.9%, 0.3%, 0.0%, 26.7%

## Выводы

1. **Теория работоспособна**, но чувствительна к параметрам.
2. RC остаётся стабильным и близок к результатам упрощённой версии.
3. Наблюдается сильная **неравномерность активности слоёв** — некоторые слои (особенно Layer 1) часто "молчат".
4. Лучшие результаты показывает стратегия `target_mode = "frequency"`.
5. Текущая теоретическая реализация даёт более высокие пиковые значения спайков в отдельных слоях, но менее сбалансирована, чем первоначальная упрощённая версия.

**Важное замечание:**  
Изначально пункт применения `resonance_function` к результату интерференции был введён сознательно, так как на момент написания теории не было понятно, какие параметры будут работать. Smoke Test должен был помочь это выяснить.

## Дальнейший план

Мы **не будем** подгонять параметры под желаемый результат. Вместо этого:

1. **Создать явную стратегию** назначения параметров (`frequency`, `threshold`, `target_mode`) для каждого фрактального уровня.
2. Документально зафиксировать эту стратегию.
3. Провести серию экспериментов, **честно** фиксируя, какие конфигурации работают, а какие — нет.
4. На основе реальных данных решить, насколько теория (в текущем виде) согласуется с практикой.
5. Только после этого переходить к Test 1.

**Следующий шаг:** Создание документа "Стратегия назначения параметров фрактальных уровней" и инструмента для честного тестирования.

**Принцип работы:** Не подгонять желаемое под действительное. Мы создаём новый тип архитектуры — нам нужны честные доказательства, а не красивые графики.
