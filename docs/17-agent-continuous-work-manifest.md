## 2. Инварианты ядра WFR (не нарушать без §5)

Пока не принято **решению о фундаментальном изменении** (§5), ядро остаётся **волново-резонансным**, а не «стеком матричного внимания».

**Сохранять по смыслу:**

1. **Wave Phase Encoder (WPE)** — фазовое кодирование позиции (в т.ч. фрактальный лог-член), не замена на стандартное учимое PE «как в GPT» без RFC.
2. **Fractal / resonance stack** — обработка через резонанс и интерференцию фаз, а не через `MultiheadAttention` / полный softmax-attention над всем контекстом как основной механизм смешивания.
3. **Spiking / пороговая нелинейность** — event-driven выбор сильных интерференций (с surrogate для градиента), а не «чистый MLP-трансформер» внутри ядра.
4. **Разделение ролей** из §10 `03-theory.md`: позиция — WPE; символ — фазовый сдвиг (embedding в фазы); **не** отождествлять токен с «одним большим `d_model` вектором в стиле Transformer» *внутри* резонансного стека.

**Допустимо в ядре без RFC** (уже в трактовке проекта): phase-locking, homeostatic пороги, multi-scale surrogate, **узкая** каузальная локальность по фазам (`phase_causal_kernel` и аналоги), если это **не** заменяет резонанс на attention.

## 3. Где граница с «обычным LLM»

### 3.1 Не считается предательством парадигмы

- **Интерфейс «поле → дискретный токен»**: `Linear` / небольшой MLP readout, `Embedding` для сдвига фаз — это **оговорённый мост к языку**, не ядро (см. §9–§11 в `03-theory.md`).
- **Сопоставимый baseline**: минимальный causal Transformer **только** как эталон в `experiments/09-lm-parity` и родственных скриптах, не как замена ядра.
- **Оптимизаторы, loss, батчи, протоколы данных** — стандартный PyTorch/ML-транспорт; это не делает WFR «обычным LLM».

### 3.2 Считается красной зоной (дрейф в обычный LLM)

Агент **не внедряет** без RFC (§5):

- Softmax self-attention (полное или с большим окном) **как основной** механизм смешивания по времени **внутри места, зарезервированного под WFR-stack**.
- Замена резонансных слоёв на «Transformer block» (Norm–Attention–FFN) под видом «улучшения».
- Глубокий стек больших FFN по токену **вместо** резонансной динамики там, где по документации должна быть волновая обработка.

Если гипотеза ровно в том, чтобы **гибридизировать** attention с WFR — это **только RFC** с явным контролем: что измеряем, как отделяем вклад WFR от вклада attention.

## 4. Цикл работы агента (почти непрерывно)

1. **Выбрать одну гипотезу** из `14` (или новую, с ID `H_next`) и **один** эксперимент, который её фальсифицирует или подтверждает.
2. **Регрессия ядра**: после правок `wfr/core.py` — минимум `experiments/00-smoke-test` (+ по времени `02` или импорт из теста Exp 09), см. `14` §1.
3. **Правка кода** — минимальный дифф под задачу; стиль и паттерны как в соседних модулях.
4. **Эксперимент**: предпочтительно скрипт в `experiments/NN-*/`, артефакты (JSON/лог) с метаданными; для LM — **fair-parity / match-capacity** где уместно (`09`).
5. **Документация**: одно из — обновить строку в матрице `14` §3 или срез `13`; при изменении теории — патч `03-theory.md` с датой и ссылкой на артефакт.
6. **Стоп-условие сессии**: записать «что сделано / что не сработало / следующий шаг» в коммите или в конец `07-experiment-plan.md` / README эксперимента — чтобы следующая сессия не начинала с нуля.

## 5. Фундаментальные новые правила (RFC)

**Когда:** ломается инвариант §2, или вводится attention/иной не-волновой **основой** стека, или меняется определение «что такое токен в поле» настолько, что §10 устаревает.

**Процедура:**

1. **RFC** — новый подраздел в `03-theory.md` (или отдельный `docs/NN-rfc-<slug>.md`): мотивация, отказ от старого инварианта, предсказания, риск для энергии/памяти/interpretability.
2. **Минимальный прототип** — изолированный эксп или флаг в существующем; старый путь остаётся переключаемым до сравнения.
3. **Число:** хотя бы один отчёт JSON/таблица; для LM — согласовать с протоколом `09`.
4. **Обновить** `00-overview.md`, `02-architecture.md` при необходимости; **добавить** строку в `14` с новым H или закрытием старого.

Без этого шага агент **не** переписывает ядро «из головы».

## 6. Приоритеты в стиле руководителя (куда класть усилия)

1. **Доказуемое преимущество** (память на длинном \(T\), энергия, стабильность continual) **на честном бенче** — выше красоты формул.
2. **Не размывать энергетическую историю**: если readout/голова раздувается до доминирования над ядром — фиксировать в `13`/`14` как риск и иметь план облегчения или аппаратного readout.
3. **Continual / 24-7** — отдельные метрики (forgetting, дрейф), не смешивать с одним только `val CE`.
