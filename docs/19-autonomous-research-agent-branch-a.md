# 19. Спецификация автономного исследовательского агента (ветка A → прорыв)

**Версия:** 1.0  
**Дата:** 4 апреля 2026  
**Назначение:** единый контракт для **долгой автономной** работы (много сессий подряд): GPU-ветка A, развитие **ядра** и **теории**, поиск новых подходов — **без** скрытого превращения проекта в обычный Transformer.

**Не является:** исполняемым ботом с бесконечным циклом в коде; это **протокол** для LLM-агента, человека-оператора GPU и оркестратора (Cursor, API, cron + ручной триггер).

**Обязательные верхние правила:** [17-agent-continuous-work-manifest.md](17-agent-continuous-work-manifest.md) — инварианты ядра и RFC.

**GPU:** режим **X** на ветке A выполняется на **удалённом** сервере по локальному `RULES.md` §§6–8 (и при необходимости `remote_sync_wikitext.ps1`); учётные данные только в `RULES.md`, не в отслеживаемых файлах.

## 1. Миссия и граница успеха («до прорыва»)

**Ветка A** в терминах репозитория: честные эксперименты на GPU (fair-parity, match-capacity где положено), артефакты в `experiments/09-lm-parity/outputs/remote_a100/runs/…`, обновление [14](14-core-readiness-and-breakthrough-matrix.md) и [13](13-project-status-snapshot.md).

**«Прорыв» для этого агента — измеримый**, не философский. Считаем, что этап A-цикла **закрыт с прорывом по выбранному критерию**, если выполнено **одно** из нижеперечисленного **с фиксацией в docs + JSON**:


| ID       | Критерий (минимум)                                                                                                                                                                                                                           | Где фиксировать                                 |
| -------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------- |
| **BR-1** | H6 закрыта: два независимых A100-прогона D.1 (разные сиды или даты) с одной и той же парой T_1 \ll T_2; таблица: CE WFR/TF, Δ, peak CUDA, wall; явное правило для TF на длинном T (полный контекст vs окно — описано в JSON/README). | 14 §3, 09-lm-parity/README, артефакты |
| **BR-2** | Энергия / память: при сопоставимом val CE WFR показывает стабильно лучший peak GPU memory или измеренные Дж/токен (если появляется протокол замера) относительно matched TF на одном корпусе и двух длинах контекста.                | новый подпункт в 03-theory или 13, артефакт         |
| **BR-3** | Ядро: новый механизм в `wfr/core.py` даёт воспроизводимое улучшение метрик Phase 0 и не ухудшает лучший известный LM-parity режим при регрессии (smoke + выбранный parity-прогон).                                                     | 03-theory, 14, PR                                   |


Пока ни один BR не выполнен — агент **не заявляет прорыв** в пользовательских текстах; пишет «гипотеза / промежуточный результат».

## 2. Режимы одного агента (смена «шляп»)

Один и тот же LLM-агент явно помечает режим в начале сессии:


| Режим              | Задача                                                                           | Выход                                                                                                                                                                                |
| ------------------ | -------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **E — Explorer**   | Литература / аналогии / новые гипотезы (физика, SSM, SNN, динамические системы). | 1–3 **фальсифицируемых** предложения в формате строки `14` §3 (гипотеза, эксперимент, метрика провала). **Без** правок ядра.                                                         |
| **T — Theorist**   | Формализация: формулы, следствия, связь с `03-theory.md`.                        | Черновик § или RFC [17](17-agent-continuous-work-manifest.md) §5; правки 03-theory только вместе с предсказанием и экспериментом. |
| **K — Core**       | Реализация в `wfr/`, минимальный дифф.                                           | Код + тест/smoke + краткое «что меняется в поле».                                                                                                                                    |
| **X — eXperiment** | Скрипты Exp 09, GPU runbook, verify-скрипты.                                     | JSON, лог, обновление `14`/`13`.                                                                                                                                                     |


Запрещено в одной сессии смешивать **K** и **X** без промежуточного **smoke** после **K**.

## 3. Жёсткое введение новой математики / фундаментальных правил

1. **Триггер:** зазор между измерением и текущим объяснением **или** необходимость сохранить инвариант при новом факте (регрессия метрики).
2. **RFC** — по [17](17-agent-continuous-work-manifest.md) §5: мотивация, что ломается/добавляется, предсказания, риски для O(1)/интерпретируемости.
3. **Минимальная формула:** одна страница в `03-theory.md` (новый подпункт) или отдельный `docs/NN-rfc-*.md` со ссылкой из `03-theory`.
4. **Двойная запись:** любая новая формула в теории должна иметь **якорь в коде** (`wfr/…`, имя класса/опции) или пометку «ещё не реализовано — эксперимент X блокирует».

Без RFC — **запрещено** менять смысл резонанса, WPE как позиционного носителя, или заменять спайковый слой на attention.

## 4. Цикл автономной сессии (повторять до BR или стоп-условия)

```
READ  → 17, 14, 13, 12 (релевантный этап), 18 при D.1
PLAN  → одна гипотеза H* или новая с ID; режим E/T/K/X
CORE? → если K: минимальный патч wfr/core.py
GATE  → python experiments/00-smoke-test/run_smoke_test.py  (и при смысле: 02 или pytest узкий)
GPU   → если X: remote шаги из RULES.md / NEXT_PHASE_RUNBOOK; один прогон за итерацию
WRITE → JSON в runs/; verify_* ; патч 14 §3 и при выводе 03/13
STOP  → отчёт в 07-experiment-plan.md или footer коммита (3–6 строк: что узнали, что дальше)
```

## 5. Поиск новых подходов (Explorer) — куда смотреть, что нельзя

**Разрешённые направления без смены миссии:**

- Усиление **представления токена во фазах** (§11.3.1 `03-theory`): каузальные смеси `embed`, не удлинение контекста через attention.
- **Readout от промежуточных слоёв** резонанса (§11.3.2): расширение наблюдаемого состояния без замены ядра.
- **Медленная поднесущая / состояние** (§11.3.3) — только после мотивации длинным T и с оговоркой по памяти в RFC.
- Стабильность: новые схемы **фазовой когерентности**, **обучаемые** частоты в узких пределах, шум/робастность — с тестом Phase 0.

**Запрещено «для скорости победы»:**

- Подмена стека Transformer-block.
- Большие FFN по всей длине как основной заменитель резонанса.
- Смешение выводов CE при разных `loss_mode` / без match-capacity.

## 6. Стоп-условия (человек или агент обязан остановиться)

- Три подряд итерации **без** нового сигнала (тот же Δ в пределах шума, те же идеи). → **Сессия E:** пересмотр постановки или смена корпуса/метрики (зафиксировать в `12`).  
- Регрессия smoke или рост CE parity **хуже контроля** после правки ядра без RFC. → **Откат** или RFC.  
- Нет доступа к GPU > N дней — только **ветка B** (`18`) и теория/T; не объявлять BR-1/BR-2.
