# 09. Тест на проверку сложности и памяти (Memory & Complexity Test)

**Версия:** 0.3  
**Дата:** 31 марта 2026  
**Статус:** Завершён (v1.0 + v2.0)  
**Цель:** Экспериментально проверить декларации теории о сложности и памяти.

## Что проверяем

Теория утверждает:
- Память на токен: **O(1)** или **O(log n)**
- Сложность по длине контекста: значительно лучше O(n) или O(n²)

Мы должны это **измерить**, а не верить на слово.

## Что будем замерять

### 1. Память (Memory Usage)
- `torch.cuda.max_memory_allocated()` / `tracemalloc` (на CPU)
- Память на **один токен** при росте контекста
- График: Memory per token vs Context Length

### 2. Время (Wall-clock time)
- Время `forward()` для разных длин контекста
- График: Time vs Context Length
- Вычисление эмпирической сложности (O(n), O(log n), O(1))

### 3. Масштабирование
- Контексты: 512, 2048, 8192, 16384, 32768, 65536, 131072, 262144
- Сравнение с теоретическими O-нотациями

## План реализации

**Файл:** `experiments/01-memory-complexity-test/run_memory_test.py`

**Что должно быть в тесте:**
- Использовать одну из лучших конфигураций из Phase 0 (например, Freq-Balanced или Mean-Balanced)
- Замерять пиковую память и время для каждого контекста
- Строить графики:
  - Memory per token vs log(context)
  - Time vs context length
  - Log-log plot для определения порядка сложности
- Сохранять результаты в `outputs/memory_complexity/`

## Сверка теории с экспериментом

| Утверждение теории | v1.0 | v2.0 | Статус |
|--------------------|------|------|--------|
| Память на токен = O(1) | 492 байта — постоянно | 492 байта — постоянно | **Подтверждено** |
| Сложность лучше O(n) | ~O(n), 1.41 с на 100M | ~O(n), 2.71 с на 100M | **Не подтверждено** (линейно) |
| RC стабилен при росте контекста | Падение 0.93 → 0.011 | Падение 0.97 → **0.46** | **Значительно улучшено** |

**Что подтверждено:**
- **O(1) память на токен** — железобетонно, на обеих версиях.
- **Phase-Locking работает**: RC на 100M вырос с 0.011 до 0.464 (×42).
- **Phase-Locking не ломает память**: потребление VRAM идентично v1.0.

**Что не подтверждено:**
- Сложность остаётся **~O(n)**, а не лучше. v2.0 даже чуть медленнее (2.71 с vs 1.41 с на 100M) из-за overhead Phase-Locking.
- RC на 100M = 0.46 — это рабочий уровень, но не идеальный. При росте контекста RC всё ещё снижается.
