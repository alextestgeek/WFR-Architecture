# 08. Phase 0 — Мастер-план проверки теории (Viability & Stability)

**Версия:** 1.0  
**Дата:** 31 марта 2026  
**Статус:** Актуален

Этот документ фиксирует **цели Phase 0**, соответствие пунктов теории ([`03-theory.md`](03-theory.md)) экспериментам, **что уже закрыто**, **что осталось** и **критерии выхода** в Phase 1 (RFP). Детальный перечень экспериментов и статусы — в [`07-experiment-plan.md`](07-experiment-plan.md).

---

## 1. Назначение Phase 0

Проверить **до обучения**:

1. Что архитектура **запускается**, даёт воспроизводимый выход и согласуется с формулами в коде (`wfr_core.py`).
2. Что декларации по **памяти и масштабу** (контекст, число слоёв) **измерены**, а не предположены.
3. Что механизмы стабильности **v2.0–v2.1** (Phase-Locking, Homeostatic, Surrogate в forward) работают в сочетании на реалистичных длинах контекста и глубине.
4. Что **внутренняя стабильность** представлений (фазы, стоячие волны, RC по окнам) при росте контекста не противоречит теории §5 (в части измеримых без задачи метрик).

**Вне Phase 0:** обучение (RFP), сравнение с трансформерами по качеству задач, энергетика «в ваттах», фотонный симулятор — см. [`05-roadmap.md`](05-roadmap.md).

## 5. Осталось в Phase 0

### 5.1 Обязательное — **выполнено**

Test 2 реализован в `experiments/04-basic-pattern-formation/`. Перед переходом к Phase 1: обновить [`05-roadmap.md`](05-roadmap.md), при необходимости — краткий абзац в [`03-theory.md`](03-theory.md) §5 со ссылкой на Experiment 04.

### 5.2 Опционально (усиливает §5 до Phase 1)

- Небольшой тест **устойчивости к шуму** (гауссов шум к фазам или к позиционным признакам) при фиксированном контексте.
- Краткая **численная сверка** реализации ключевых формул (резонанс, интерференция) с эталоном в тесте — по желанию, не блокирует Phase 1.

## 6. Критерии завершения Phase 0

Phase 0 считается **завершённой** для перехода к Phase 1 (RFP), если:

1. Пункт **5.1** (Test 2) выполнен — **да** (Experiment 04); при желании добавить абзац в [`03-theory.md`](03-theory.md) §5.
2. [`07-experiment-plan.md`](07-experiment-plan.md) обновлён: Test 2 — «Завершён».
3. [`05-roadmap.md`](05-roadmap.md) отражает закрытие Phase 0 и переход к обучению.
4. Ссылки в [`docs/README.md`](README.md) и [`00-overview.md`](00-overview.md) согласованы.

**Не являются блокером:** провал отдельного опционального пункта из §5.2 при явной фиксации в отчёте.

## 7. Известные ограничения (зафиксировать при закрытии Phase 0)

- Время forward **~O(n)**, не суб-линейно — см. [`09-memory-complexity-test-plan.md`](09-memory-complexity-test-plan.md).

- Заявления [`04-comparison.md`](04-comparison.md) про энергию и железо остаются **качественными** до отдельных измерений.
